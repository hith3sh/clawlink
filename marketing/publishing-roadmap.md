# ClawLink Publishing Roadmap

## Objective
Grow attention for ClawLink by publishing a steady stream of high-intent, issue-driven, and integration-specific content that OpenClaw users can actually discover and use.

## Positioning rule
Every page should answer one of these:

- I hit an error — how do I fix it?
- I want to connect X — how do I do it?
- I’m confused about how this OpenClaw thing works — explain it clearly.
- I want a useful workflow — show me one.

If a page does none of those, it is probably content debt.

## Priority buckets

### Bucket A — highest ROI now
- exact error pages
- ACPX/ACP confusion pages
- Gmail / Calendar / HubSpot / Slack connection pages

### Bucket B — next highest
- workflow pages combining integrations
- cost reduction / token usage
- update regression troubleshooting

### Bucket C — authority builders
- support matrix pages
- plugin troubleshooting hubs
- device pairing / architecture explainers

---

## Distribution plan per article
For each published article:

1. Add at least 3 internal links from related docs/pages.
2. Reference it in future support replies.
3. Share it in relevant communities only when it genuinely answers the question.
4. Add it to a docs or troubleshooting index page.
5. Reuse it as source material for short-form posts.

---

## Success criteria
Within 30 days, the site should have:

- at least 12–18 new useful pages
- clear clusters around troubleshooting + integrations
- enough internal linking to guide users toward setup
- multiple exact-problem pages that can rank for long-tail searches

## Operating principle
The fastest path to attention is not broad marketing.
It is becoming the page people find when they are blocked, confused, or trying to connect something right now.
